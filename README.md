## Overview

Stage203 upgrades the Mini-PoC into an **external-review-ready package**:

- **One-command execution**: `INSTALL_DEPS=1 ./run_poc.sh`
- **Evidence verification** (fail-closed) written into `out/evidence/**/result.txt`
- **PoC report generation**: `out/reports/poc_report.md`
- **External review intake** via GitHub Issue template

This stage is not a theoretical document.  
It is a **reproducible evidence chain**.

What This PoC Demonstrates

Stage203 demonstrates that QSP claims are:

Explicitly declared

Bound to required checks

Linked to concrete evidence artifacts

Verifiable in a closed environment (no special infra required)

Architecture Principle

Stage203 enforces:

Fail-closed semantics

Explicit security claims (A2–A5)

Verification bound to evidence artifacts

No implicit trust assumptions

If required evidence is missing or verification fails, the run must fail closed.

Evidence Chain (What We Verify)

Stage203’s mini_poc scripts verify the evidence chain in a minimal, practical way:

attack_replay

Parses out/poc_logs/poc.jsonl

Requires stage191_ci_summary to include:

job attack_replay with conclusion=success

Writes: out/evidence/attack_replay/result.txt

attack_downgrade

Parses out/poc_logs/poc.jsonl

Requires stage191_ci_summary to include:

job attack_downgrade with conclusion=success

Writes: out/evidence/attack_downgrade/result.txt

attack_drift_injection

Until a dedicated CI job exists, we require the PoC to have reached:

event claim_gate_passed

Writes: out/evidence/attack_drift_injection/result.txt

This keeps the chain explicit and auditable, even before full automation exists.

Non-goals (Explicitly Not Claimed)

Stage203 does not claim:

Unconditional security

QKD guarantees

Formal proof completeness

Real multi-node VPS deployment (this is a closed-environment PoC)

Instead, it enforces:

Claim transparency

Execution-level verifiability

Evidence visibility

Fail-closed behavior on missing evidence

Claims Structure

Declared in:

claims/claims.yaml

Each claim contains:

required_jobs

evidence_paths

Example:

claims:
  A2:
    required_jobs: ["attack_replay"]
    evidence_paths:
      - "out/evidence/attack_replay/result.txt"
